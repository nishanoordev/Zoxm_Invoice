# 📄 PRODUCT REQUIREMENTS DOCUMENT (PRD)

## 🧾 Product Name

Zoxm_Invoice – Shop Management App

---

## 🎯 Objective

Develop a secure, offline-first mobile application that enables small businesses to manage:

* Customers
* Invoices
* Inventory
* Payments
* Delivery Challans
* Cloud Sync (Firebase)

---

## 👤 Target Users

* Small shop owners
* Retail & wholesale businesses
* Individual entrepreneurs

---

## 📱 Platform & Tech Stack

* Frontend: React Native (Expo)
* Backend: Firebase Realtime Database
* Authentication: Firebase Auth
* Storage: Local (AsyncStorage/SQLite) + Cloud Sync

---

## 🧩 Core Modules & Requirements

---

### 1. Dashboard

#### Features

* Display key metrics:

  * Total invoices
  * Paid invoices
  * Pending invoices
  * Overdue invoices
  * Total revenue
* Show recent invoices list
* Provide quick actions:

  * Create invoice
  * Add customer
  * Add item

#### Functional Requirements

* Data must be dynamically calculated from stored invoices
* Dashboard must update in real-time after changes

---

### 2. Customers Module

#### Features

* List customers with filters:

  * Active
  * Unpaid
  * All
* Display:

  * Name
  * Email
  * Receivables
  * Unused credits
* Add, edit, delete customer

#### Customer Details

* Tabs:

  * Details
  * Transactions
  * Comments & History
* Show:

  * All invoices linked to customer
  * Payment history
  * Activity timeline

#### Functional Requirements

* Receivables = sum of unpaid invoice balances
* Transactions must reflect real-time invoice/payment data

---

### 3. Invoice Module

#### Features

* Create, edit, delete invoices
* Add multiple line items
* Select rate type:

  * Retail
  * Wholesale
* Apply:

  * Tax (%)
  * Discount (%)
* Status:

  * Draft
  * Sent
  * Paid
  * Overdue

#### Filters

* Unpaid / Paid / Draft / All

#### Functional Requirements

* Automatic calculation:

  * Subtotal
  * Tax
  * Discount
  * Total
* Automatic status update based on:

  * Payment
  * Due date

---

### 4. Inventory Module

#### Features

* Add and manage items
* Fields:

  * Name
  * SKU
  * Retail price
  * Wholesale price
  * Quantity
  * Unit
  * Category
* Generate QR code for items

#### Functional Requirements

* Stock value must be calculated dynamically
* Items must be selectable in invoice creation

---

### 5. Payments Module

#### Features

* Record payments
* Filter:

  * All
  * Invoice-based
* Display:

  * Customer name
  * Amount
  * Date
  * Payment method

#### Functional Requirements

* Payments must be linked to invoices
* Invoice balance must update automatically

---

### 6. Delivery Challan Module

#### Features

* Create delivery challan
* Status:

  * Draft
  * Open
  * Delivered
* Link challan with invoice

---

### 7. Comments & History

#### Features

* Maintain activity timeline for:

  * Customer creation
  * Invoice creation
  * Status updates
  * Payments
* Display chronological logs

#### Functional Requirements

* Each action must be recorded with timestamp
* Logs must be immutable

---

### 8. Sync to Cloud

#### Features

* Button: “Sync to Cloud” in Settings
* Upload all local data to Firebase

#### Flow

1. User clicks sync
2. Show loading indicator
3. Upload data
4. Show success or error popup

#### Functional Requirements

* Sync must be user-specific (UID-based)
* No data overwriting between users
* Must handle failures and retries

---

### 9. Settings Module

#### Features

* Email preferences
* Privacy & Security
* Feedback
* Share app
* Rate app
* About
* Logout
* Sync to Cloud

---

### 10. Navigation

#### Sidebar Menu

* Home
* Customers
* Items
* Quotes
* Delivery Challans
* Invoices
* Payments
* Reports
* Settings

---

## 🗄️ Data Requirements

* All data must be stored per user using unique UID
* Data structure must include:

  * Profile
  * Customers
  * Items
  * Invoices
  * Payments
  * Challans
  * History

---

## 🔐 Security Requirements

* Use Firebase Authentication
* Restrict database access per user
* Prevent unauthorized reads/writes
* Do not store sensitive data locally without protection

---

## ⚠️ Error Handling

* Handle:

  * No internet connection
  * Sync failures
  * Invalid inputs
* Provide user-friendly error messages
* Allow retry actions

---

## 📡 Offline Support

* App must work without internet
* Store data locally
* Sync when connection is available

---

## 🚀 Future Enhancements

* Multi-user access (team accounts)
* Automatic cloud sync
* Sales reports and analytics
* Export CSV / PDF
* Overdue reminders and notifications

---

## ✅ Acceptance Criteria

* User can manage customers, invoices, and inventory
* Invoice calculations are accurate
* Payments correctly update balances
* Dashboard reflects real-time data
* Sync works without data loss
* Data is securely stored per user
* App functions offline and syncs later
